## Sample Data Layout
The target CSV log (`samples\colesdata.csv`) format:
- **Row 1**: Metadata (e.g., `Report Generated: 29/05/2026      06:06:22 PM (AEST),,,,,,,`) - **Must be skipped.**
- **Row 2**: Headers - **Must be skipped or used as keys.**
- **Row 3+**: Data rows

### Explicit Column Names to Parse
- `Event`
- `To`
- `Date` (Format: MM/DD/YYYY)
- `Time`
- `Duration (Sec)`
- `Data Up (KB)`
- `Data Down (KB)`
- `Price`

## Active State Framework
The application has three primary View Modes:
1. **Raw**: Full unaggregated list of all log entries.
2. **Daily**: Grouped and sorted strictly by day. Aggregates data usage, duration, etc.
3. **Monthly**: Grouped by Month/Year (e.g., May 2026). 
   - **Monthly -> Daily Drill-down Toggle**: Clicking any Month row expands a nested, clean table showing the Daily Breakdown for just that specific month.

## Design System & Theme
- **SaaS Dark Mode**: The UI should utilize a gorgeous dark theme specifically using Tailwind classes: `bg-zinc-950`, `text-zinc-50`, `border-zinc-800`.
- **Table Capabilities**: Clickable header sorting (handling string vs mathematical numeric sorting automatically), sticky table headers, and a global search/filter bar for all views.
